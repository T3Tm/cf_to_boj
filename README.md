# 🛠️ Codeforces 백준화 확장 프로그램 통합 명세서 (v1.4)

본 문서는 코드포스를 백준(BOJ/solved.ac) 스타일로 변환하기 위해 설계된 시스템 아키텍처, 검색 쿼리 엔진, 그리고 **렌더링 성능 최적화 및 브라우저 환경 제어 로직**을 총망라한 최종 명세입니다.

---

## 1. 레이팅 및 티어 시스템 (100점 단위 계단식 설계)

### ① 유저/문제 티어 매핑
- **유저:** CF 색상 기준 백준 티어 매핑. 마스터(2400+) 상한 및 비로그인 예외 처리.
- **문제:** 100점 당 1단계 상승 공식 적용 (브론즈 5 ~ 루비 3). 800점~3500점 구간.

---

## 2. 통합 검색 필터링 엔진 (solved.ac 문법 이식)

### ① 검색 문법 및 파이프라인
- **지원 조건:** `id:`, `tag:#`, `tier:*`, `rating:r:`, 상태 필터(`s@me` 등).
- **DOM 필터링(Execution) 아키텍처:**
  1. 트리거 발생 시 현재 활성화된 알약(Pill) 조건 배열 수집.
  2. 테이블의 각 문제 행(`tr`)을 순회하며 메타데이터 추출.
  3. 모든 활성 조건을 교집합(AND)으로 만족하는 행만 표출(`display: table-row`).

### ② [신규] 렌더링 성능 최적화 (Anti-Reflow)
- 수천 개의 DOM 요소를 직접 조작할 때 발생하는 브라우저 병목(Freezing) 방지를 위해, 메모리 상에서 CSS 클래스 기반으로 가시성 상태를 일괄 계산 후 화면에 1회 반영하는 일괄 처리(Batching) 로직 적용.

---

## 3. 예외 상황 및 UI 제어 (State Management)

### ① 검색 결과 없음 (Empty State) 처리
- 활성화된 필터 조건을 만족하는 문제가 0개일 경우, 빈 테이블 대신 "조건과 일치하는 문제가 없습니다"라는 안내 메세지를 동적으로 생성하여 렌더링.

### ② 페이지네이션 동적 억제
- 필터 조건(알약)이 하나라도 활성화된 상태에서는 코드포스의 기본 하단 페이지네이션 노출을 강제 차단. 필터가 모두 해제되면 페이지네이션을 원상 복구하여 필터 붕괴(State Loss) 현상 방지.

---

## 4. 페이지별 레이아웃 개조 사항

### ① 공통 네비게이션 (GNB)
- 4대 핵심 탭 단순화 및 **수동 테마 토글러(Light/Dark 스위치)** 배치.

### ② 문제 목록 (Problemset)
- 개인화 컬러링(맞은 문제/틀린 문제) 적용 및 상단 페이지네이션 제거를 통한 여백 확보.

---

## 5. 개별 문제 페이지 (Problem Detail) 방어

### ① 태그 스포일러 원천 차단 로직
- 코드포스 기본 태그 렌더링 강제 차단 후, 지문 하단에 독립된 백준 스타일 '보기/가리기' 토글 UI 주입.

---

## 6. 프로필 및 통신 충돌 방어

### ① API Rate Limit 우회 (로컬 캐싱)
- `user.status` API 호출 시 `localStorage`에 15분 단위로 캐싱하여 429 에러 방지. 수동 데이터 갱신 버튼 제공.

---

## 7. 글로벌 환경 변수 및 동적 렌더링 방어

### ① 커스텀 테마 제어 시스템 (Theme Engine)
- **독립 스코프:** 시스템 미디어 쿼리 의존성 탈피. HTML `data-theme` 속성으로 제어.
- **깜빡임(FOUC) 방어:** 페이지 로드 시 UI가 렌더링되기 전에 `localStorage`의 테마 값을 최우선으로 읽어와 화면 깜빡임 없이 다크모드 적용.

### ② AJAX DOM 증발 방어 (MutationObserver)
- 서버 비동기(AJAX) 통신으로 테이블이 갱신될 때마다 `MutationObserver`가 이를 감지하여 티어 아이콘 렌더링 및 필터 상태 자동 복구.
