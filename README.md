## 🚨 2. 치명적 버그 원인 분석 및 해결 설계

### ① 라우팅 붕괴 (Submit/Status 페이지 검색창 침범)
* **원인 파악:** 진입점(main.js)에서 URL을 검사할 때, 단순 문자열 포함 여부(includes('/problemset'))를 사용했습니다. 이로 인해 '/problemset/submit'과 '/problemset/status'에도 문제 목록용 컨트롤러가 실행되어 검색창이 잘못 주입되었습니다.
* **해결 설계:** 라우팅 로직을 '정규 표현식(Regex)' 또는 '정확한 일치(Exact Match)' 검사로 전면 수정합니다. 문제 목록은 오직 정확한 '/problemset' 또는 '/problemset/page/X' 구조일 때만 실행되도록 밸브를 조입니다.

### ② 타겟팅 오류 (Status 페이지 문제 번호 아이콘 오적용)
* **원인 파악:** 문제 목록 테이블과 채점 현황 테이블은 구조가 다릅니다. 채점 현황의 첫 번째 열은 '제출 번호(Submission ID)'인데, 기존 로직이 무조건 테이블의 첫 번째 열에 티어 아이콘을 박아 넣도록 설계되어 발생한 대참사입니다.
* **해결 설계:** 신규 'status.js' 컨트롤러를 분리하여 생성합니다. 채점 현황 페이지에서는 검색창 생성을 차단하고, DOM 탐색 로직을 수정하여 제출 번호가 아닌 'Problem(문제 이름)'이 위치한 특정 열(Column)을 정확히 타겟팅하여 티어 아이콘을 부여합니다.

### ③ 다크모드 텍스트 시인성 및 CSS 충돌
* **원인 파악:** 상단 헤더 메뉴(HOME, TOP, CATALOG 등)와 특정 본문 텍스트들이 코드포스의 고유 클래스로 강력하게 묶여 있어, 우리가 주입한 다크모드 배경색 위에서 검은색 글씨로 남아버렸습니다.
* **해결 설계:** 'overrides.css'의 다크모드 선택자 범위를 대폭 확장합니다. 단순히 배경만 바꾸는 것을 넘어, 헤더의 메뉴 리스트, 테이블 헤더, 범용 텍스트 컨테이너의 글자색을 명시도가 매우 높은 방식(High Specificity)으로 밝은 색상 변수로 강제 덮어씌웁니다.

### ④ 문제 페이지(Problem) 백준 스타일 미적용
* **원인 파악:** 개별 문제 페이지 우측에 여전히 코드포스 기본 사이드바(Pay attention 등)가 남아있어 백준 특유의 깔끔한 1단 레이아웃 느낌이 살지 않았습니다.
* **해결 설계:** 'problem.js' 컨트롤러가 실행될 때, 우측 사이드바 컨테이너를 DOM에서 강제로 숨김 처리(display: none)하고, 본문 영역(Problem Statement)의 가로 폭을 화면 중앙으로 넓혀 백준과 동일한 시각적 레이아웃을 구성하도록 로직을 추가합니다.

## 🛡️ 3. 자체 교차 검증 (3-Pass Verification)

수정된 설계에 대해 논리적 허점이 없는지 3회 검증한 결과입니다.

* **1차 검증 (라우팅 충돌):** '/problemset' 패스와 '/problemset/status' 패스가 완벽히 분리되는가? -> 라우터에서 URL 문자열 분할 및 완전 일치 연산자를 도입하여 교집합 오류를 100% 차단함.
* **2차 검증 (DOM 옵저버 과부하):** 신규 생성된 status.js에서도 DOM 옵저버가 필요한가? -> 채점 현황 역시 비동기로 갱신되므로 DOMObserver를 연결하되, 대상 컨테이너를 정확히 현황판 테이블로 지정하여 무한 루프를 방지함.
* **3차 검증 (다크모드 우선순위):** 코드포스가 향후 CSS를 업데이트해도 다크모드가 유지되는가? -> 최상위 HTML 태그의 data-theme 속성을 기준으로 하위의 모든 a, p, span, li 태그에 색상 변수를 상속시키도록 설계하여 레거시 업데이트에 대한 방어력을 갖춤.

## 📝 4. 적용 가이드 (Next Steps)

위 설계를 바탕으로 실제 코드를 수정하실 때는 다음 순서를 권장합니다.

1.  가장 먼저 'main.js'의 라우터 조건문을 뜯어고쳐서 페이지별 컨트롤러가 엉뚱한 곳에서 실행되지 않도록 막으십시오.
2.  'pages/' 폴더 내에 'status.js'를 새로 만들고, 표의 3번째 또는 4번째 열(문제 이름)에만 아이콘을 넣는 로직을 작성하십시오.
3.  'overrides.css'를 열어 다크모드 속성 블록 안에 헤더 메뉴용 CSS 선택자를 추가하십시오.
