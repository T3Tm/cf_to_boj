# 🛠️ Codeforces 백준화 확장 프로그램 통합 명세서 (v1.3)

본 문서는 코드포스를 백준(BOJ/solved.ac) 스타일로 변환하기 위해 설계된 시스템 아키텍처, 검색 쿼리 엔진, 그리고 테마(Theme) 제어 및 클라이언트 사이드 필터링 로직을 총망라한 최종 명세입니다.

---

## 1. 레이팅 및 티어 시스템 (100점 단위 계단식 설계)

### ① 유저/문제 티어 매핑
- **유저:** CF 색상을 기준(Newbie~Legendary)으로 백준 티어 매핑 (2400점 이상 마스터 고정).
- **문제:** 100점 당 1단계 상승 공식 적용 (브론즈 5 ~ 루비 3). 800점~3500점 구간.

---

## 2. 통합 검색 필터링 엔진 (solved.ac 문법 이식)

### ① 검색 문법 및 파이프라인
- **지원 조건:** `id:`, `tag:#`, `tier:*`, `rating:r:`, 상태 필터(`s@me` 등).
- **DOM 필터링(Execution) 아키텍처:**
  1. 트리거 발생 시 현재 활성화된 알약(Pill) 조건 배열 수집.
  2. 테이블의 각 문제 행(`tr`)을 순회하며 메타데이터(티어, 태그, 제목) 추출.
  3. 모든 활성 조건을 교집합(AND)으로 만족하는 행만 노출(`display: table-row`), 나머지는 숨김(`display: none`) 처리.

### ② UI 및 상호작용
- 키보드(`Enter`, `,`) 및 마우스 클릭 이벤트 통합으로 알약 UI 동적 생성.

---

## 3. 페이지별 레이아웃 개조 사항

### ① 공통 네비게이션 (GNB)
- 메뉴를 4대 핵심 탭으로 단순화 및 우측 드롭다운 구현.
- **[신규] 다크모드 토글러:** 유저 닉네임 좌측에 수동 테마 변경(Light/Dark) 토글 스위치 배치.

### ② 문제 목록 (Problemset)
- 개인화 컬러링(맞은 문제 초록, 틀린 문제 빨강 적용). 상단 페이지네이션 제거 및 여백 확보로 검색창 레이아웃 충돌 원천 차단.

---

## 4. 개별 문제 페이지 (Problem Detail) 충돌 방어

### ① 태그 스포일러 원천 차단 로직
- 코드포스 기본 태그 렌더링 강제 차단. 지문 하단에 독립된 백준 스타일 '보기/가리기' 토글 UI 주입으로 스포일러 방지.

---

## 5. 프로필 및 통신 충돌 방어 아키텍처

### ① API Rate Limit 우회 (로컬 캐싱)
- `user.status` API 호출 시 `localStorage`에 15분 단위로 타임스탬프와 함께 캐싱하여 429 에러(Too Many Requests) 방지. 수동 데이터 갱신 버튼 제공.

---

## 6. 글로벌 환경 변수 및 동적 렌더링 방어

### ① 커스텀 테마 제어 시스템 (Theme Engine)
- **독립 스코프:** 시스템의 `prefers-color-scheme` 미디어 쿼리 의존성 탈피.
- **상태 관리:** HTML 태그의 커스텀 속성(`data-theme`)을 통해 라이트/다크 테마를 제어하며, 토글 상태는 `localStorage`에 영구 저장하여 페이지 이동 시에도 일관된 테마 유지.

### ② AJAX DOM 증발 방어 (MutationObserver)
- 문제 목록 표가 서버 비동기(AJAX) 통신으로 갱신될 때마다 `MutationObserver`가 이를 감지하여 티어 아이콘 렌더링 함수를 0.001초 내로 재실행.
