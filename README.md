# 🛠️ Codeforces 백준화 확장 프로그램 통합 명세서 (v1.1)

본 문서는 코드포스를 백준(BOJ/solved.ac) 스타일로 변환하기 위해 설계된 모든 UI/UX 규칙, 시스템 아키텍처 및 트러블슈팅 내역을 정리한 최종 명세입니다.

---

## 1. 레이팅 및 티어 시스템 (100점 단위 계단식 설계)

### ① 유저 티어 (User Rating)
- 코드포스의 유저 색상(Newbie~Legendary Grandmaster)을 기반으로 백준 티어와 매핑.
- 2400점 이상은 공통적으로 `master.svg` 부여.
- 로그인하지 않은 상태(Guest)는 `norating.svg` 처리.

### ② 문제 티어 (Problem Difficulty)
- **규칙:** 100점 당 1단계 상승 (선택지 A 적용)
- **범위:** 800점(Bronze 5) ~ 3500점(Ruby 3)
- **상한선:** 3500점을 초과하더라도 루비 3으로 고정.
- **예외 처리:** 난이도 정보가 없거나(`?`) 0점인 경우 `question_mark.svg` 노출.

## 2. 통합 검색 필터링 엔진 (solved.ac 문법 이식)

### ① 검색 문법 (Syntax)
- **기본 접두사:** `id:`, `tag:#`, `tier:*`, `rating:r:`, `s#` (해결 수).
- **상태 필터:** `s@me` (맞은 문제), `~s@me` (안 푼 문제), `t@me` (시도한 문제).
- **논리 연산:** `|` (OR), `~` (NOT), `()` (우선순위 그룹화), `공백` (AND).
- **티어 범위 검색:** `tier:b5..s1` 입력 시 자동으로 `800..1700` 레이팅 범위로 변환.

### ② UI 및 이벤트 로직 (Event Handling)
- **키보드 이벤트:** `Enter` 또는 `,`(쉼표) 입력 시 텍스트를 알약(Pill) UI로 변환. `Backspace`로 최근 알약 삭제.
- **마우스 이벤트:** [검색] 버튼 클릭 시 입력창의 잔여 텍스트를 알약으로 강제 변환 후, 수집된 모든 필터 조건을 바탕으로 필터링 엔진 트리거.

### ③ 실행 아키텍처
- **SPA 렌더링:** 코드포스 서버에 의존하지 않고, 전체 문제 데이터를 메모리에 캐싱하여 브라우저 단에서 즉시 필터링.

## 3. 페이지별 레이아웃 개조 사항

### ① 공통 네비게이션 (GNB)
- 메뉴를 [문제, 대회, 채점 현황, 랭킹] 4대 메뉴로 단순화.
- 우측 상단 유저 드롭다운 메뉴(내 정보, 설정, 로그아웃) 구현.

### ② 문제 목록 (Problemset)
- **태그 스포일러 방지:** 테이블 내의 모든 알고리즘 태그 영역 영구 숨김.
- **개인화 컬러링:** 맞은 문제(초록), 틀린 문제(빨간색) 강조. (행 배경색 삭제)
- **정렬 UI:** 난이도 및 해결 수 헤더 클릭 시 서버 정렬 연동 및 화살표 표시.

### ③ 개별 문제 페이지 (Problem Detail)
- **알고리즘 분류 토글:** 사이드바 태그 숨김 및 지문 하단 '보기/가리기' 버튼으로 스포일러 차단.
- **예제 복사:** 입출력 데이터 우측 클립보드 복사 버튼 추가.

## 4. 안정성 및 버그 수정 (Hotfix - 완료)

### ① 자바스크립트 런타임 안전장치
- `getCurrentUser` 복구: 페이지 내 유저 정보 존재 여부를 검사하고 `null`을 안전하게 반환하여 스크립트 중단(Crash) 방지.

### ② 레이아웃 충돌 및 검색 이벤트 해결
- 검색창과 테이블 겹침 방지를 위해 상단 페이지네이션 제거 및 `clear: both` 적용.
- 알약 과다 생성 시 최대 높이 제한 및 내부 스크롤(`max-height`, `overflow-y`) 적용.
- 검색 버튼 마우스 클릭(`click`) 이벤트 리스너 누락 해결.

## 5. 잠재적 충돌 방어 로직 (Future Defenses - 신규)

### ① API Rate Limit (초당 호출 제한) 방어
- **상황:** 프로필 페이지 등에서 `user.status` API를 무분별하게 호출 시 차단 위험.
- **대응 설계:** API 호출 전 `localStorage`나 `sessionStorage`에 데이터를 단기 캐싱(약 10분)하여, 불필요한 중복 호출을 막고 로딩 속도 최적화.

### ② 다국어(러시아어) 및 다크 모드 충돌 방어
- **상황:** 유저 설정 변경 시 텍스트 파싱 오류 및 UI 색상 붕괴 위험.
- **대응 설계:** 텍스트 기반 크롤링 대신 DOM 구조(nth-child 등) 기반으로 데이터를 추출하도록 수정. 주입되는 CSS에 다크 모드용 미디어 쿼리(`@media (prefers-color-scheme: dark)`) 또는 코드포스 다크 모드 클래스(`.dark-theme`) 호환 스타일 추가.

### ③ AJAX 통신 (MutationObserver 도입)
- **상황:** 페이지 이동 없이 테이블 내용 갱신 시 티어 아이콘 초기화.
- **대응 설계:** `MutationObserver`를 통해 테이블 DOM 변화를 감지하고 렌더링 함수 자동 재실행.
