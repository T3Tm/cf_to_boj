# 🛠️ Codeforces 백준화 확장 프로그램 통합 명세서 (v1.2)

본 문서는 코드포스를 백준(BOJ/solved.ac) 스타일로 변환하기 위해 설계된 시스템 아키텍처, 검색 쿼리 엔진, 그리고 **각종 환경적 충돌을 방어하기 위한 최신 렌더링 및 캐싱 방어 로직**을 총망라한 최종 명세입니다.

---

## 1. 레이팅 및 티어 시스템 (100점 단위 계단식 설계)

### ① 유저 티어 (User Rating)
- 코드포스의 유저 색상(Newbie~Legendary Grandmaster)을 기반으로 백준 티어와 매핑.
- 2400점 이상은 공통적으로 `master.svg` 부여. 비로그인은 `norating.svg` 처리.

### ② 문제 티어 (Problem Difficulty)
- **규칙:** 100점 당 1단계 상승 (브론즈 5 ~ 루비 3). 상한선 3500점(루비 3) 고정.
- **예외 처리:** 난이도 정보 부재 시 `question_mark.svg` 노출.

---

## 2. 통합 검색 필터링 엔진 (solved.ac 문법 이식)

### ① 검색 문법 및 파이프라인
- **지원 조건:** `id:`, `tag:#`, `tier:*`, `rating:r:`, `s#`(해결 수), 상태(`s@me`, `~s@me`, `t@me`).
- **논리 연산:** `|` (OR), `~` (NOT), `()` (그룹화), `공백` (AND).
- **실행 아키텍처:** 전체 문제 데이터를 메모리에 캐싱하여 브라우저 단에서 독립적(SPA)으로 즉시 필터링 수행. 서버 쿼리 의존도 0%.

### ② UI 및 상호작용
- 키보드(`Enter`, `,`) 및 마우스 클릭([검색] 버튼) 이벤트를 통합하여 알약(Pill) UI 생성 및 필터 트리거.

---

## 3. 페이지별 레이아웃 개조 사항

### ① 공통 네비게이션 (GNB)
- 메뉴를 [문제, 대회, 채점 현황, 랭킹] 4대 핵심 탭으로 개편 및 우측 프로필 드롭다운 구현.

### ② 문제 목록 (Problemset)
- 개인화 컬러링(맞은 문제 초록, 틀린 문제 빨강 적용). 표 배경색 제거를 통한 시인성 강화.
- 레이아웃 충돌 방지를 위해 상단 페이지네이션 제거 및 독립된 `clear: both` 블록 형성.

---

## 4. 개별 문제 페이지 (Problem Detail) 충돌 방어

### ① 태그 스포일러 원천 차단 로직
- **문제점:** 우측 사이드바 노출 및 기존 토글 로직 DOM 붕괴 위험.
- **해결책:** 코드포스 기본 태그 렌더링을 CSS로 강제 차단. 문제 지문 하단에 독립적인 커스텀 컴포넌트(백준 스타일 Toggle Box)를 주입하여 스포일러 방지 및 안정성 확보.

---

## 5. 프로필(Profile) 및 통신 충돌 방어 아키텍처

### ① API Rate Limit (초당 호출 제한) 우회 캐싱 시스템
- **문제점:** 프로필 접근 시 잦은 API(`user.status`) 호출로 인한 서버 차단 방지.
- **해결책:** 1. 최초 1회 호출 후 `localStorage`에 데이터 및 타임스탬프 저장.
  2. 15분 이내 재접속 시 캐시 데이터 즉시 렌더링 (호출 생략).
  3. UI에 수동 **[↻ 데이터 갱신]** 버튼을 제공하여 유저 통제권 부여.

---

## 6. 글로벌 환경 변수 충돌 방어 (Global Defenses)

### ① 다크 모드(Theme) 및 다국어(Language) 호환성
- **테마 방어:** 주입되는 모든 커스텀 UI(검색창, 프로필 카드 등)에 고정 색상 대신 동적 CSS 변수를 매핑하여, 코드포스 다크 모드 전환 시 UI가 어색하게 뜨는 시각적 붕괴 방지.
- **다국어 방어:** "러시아어" 등 설정 시 텍스트 파싱 오류를 막기 위해, DOM 텍스트 크롤링 방식을 전면 폐기하고 DOM 구조(nth-child) 및 전역 객체 상태 기반의 데이터 추출 방식으로 전환.

### ② AJAX DOM 증발 방어 (MutationObserver)
- **문제점:** 페이지네이션 및 정렬 시 표 부분만 AJAX로 덮어씌워져 주입된 티어 아이콘이 소멸됨.
- **해결책:** `MutationObserver` 인스턴스를 생성하여 `#pageContent` 하위 요소의 변경을 실시간 감시. 렌더링 갱신이 감지될 때마다 커스텀 아이콘 렌더링 및 텍스트 번역 함수를 자동 재호출하여 상태 유지.
