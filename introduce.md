# 🛠️ Codeforces 백준화 확장 프로그램 통합 명세서

본 문서는 코드포스를 백준(BOJ/solved.ac) 스타일로 변환하기 위해 설계된 모든 UI/UX 규칙 및 시스템 아키텍처를 정리한 최종 명세입니다.

---

## 1. 레이팅 및 티어 시스템 (100점 단위 계단식 설계)

### ① 유저 티어 (User Rating)
- 코드포스의 유저 색상(Newbie~Legendary Grandmaster)을 기반으로 백준 티어와 매핑.
- 2400점 이상은 공통적으로 `master.svg` 부여.
- 로그인하지 않은 상태(Guest)는 `norating.svg` 처리.

### ② 문제 티어 (Problem Difficulty)
- **규칙:** 100점 당 1단계 상승 (선택지 A 적용)
- **범위:** 800점(Bronze 5) ~ 3500점(Ruby 3)
- **상한선:** 3500점을 초과하더라도 루비 3으로 고정.
- **예외 처리:** 난이도 정보가 없거나(`?`) 0점인 경우 `question_mark.svg` 노출.

## 2. 통합 검색 필터링 엔진 (solved.ac 문법 이식)

### ① 검색 문법 (Syntax)
- **기본 접두사:** `id:`, `tag:#`, `tier:*`, `rating:r:`, `s#` (해결 수).
- **상태 필터:** `s@me` (맞은 문제), `~s@me` (안 푼 문제), `t@me` (시도한 문제).
- **논리 연산:** `|` (OR), `~` (NOT), `()` (우선순위 그룹화), `공백` (AND).
- **티어 범위 검색:** `tier:b5..s1` 입력 시 자동으로 `800..1700` 레이팅 범위로 변환하여 필터링.

### ② 실행 아키텍처
- **SPA 렌더링:** 코드포스 서버에 의존하지 않고, 전체 문제 데이터를 메모리에 캐싱하여 브라우저 단에서 즉시 필터링.
- **UI 피드백:** 입력된 쿼리를 해석하여 검색창 하단에 알약(Pill) 모양의 태그 시각화 제공.

## 3. 페이지별 레이아웃 개조 사항

### ① 공통 네비게이션 (GNB)
- 메뉴를 [문제, 대회, 채점 현황, 랭킹] 4대 메뉴로 단순화.
- 우측 상단 유저 드롭다운 메뉴(내 정보, 설정, 로그아웃) 구현.

### ② 문제 목록 (Problemset)
- **태그 스포일러 방지:** 테이블 내의 모든 알고리즘 태그 영역을 영구 숨김 처리.
- **개인화 컬러링:** 내가 맞은 문제는 제목을 **초록색**, 틀린 문제는 **빨간색**으로 강조. (행 배경색은 삭제하여 가독성 확보)
- **정렬 UI:** 난이도 및 해결 수 헤더 클릭 시 서버 정렬 연동 및 화살표(▲/▼) 표시.

### ③ 개별 문제 페이지 (Problem Detail)
- **알고리즘 분류 토글:** 기본적으로 숨김 상태이며, '보기' 버튼 클릭 시에만 노출.
- **예제 복사:** 입출력 데이터 우측에 클릭 시 클립보드로 복사되는 편의 기능 추가.

## 4. 안정성 및 버그 수정 (Hotfix)

### ① 자바스크립트 런타임 안전장치
- **getCurrentUser 복구:** 페이지 내 유저 정보 존재 여부를 검사하고 `null`을 안전하게 반환하여 스크립트 중단(Crash) 방지.
- **연쇄 오류 차단:** 함수 호출 실패 시 뒤따르는 레이아웃 조정 로직이 취소되지 않도록 로직 분리.

### ② 레이아웃 충돌 해결
- 검색창과 테이블이 겹치는 현상을 방지하기 위해 상단 페이지네이션 제거 및 `clear: both` 블록 분리 적용.
- 검색 알약 과다 생성 시 메인 컨텐츠를 밀어내지 않도록 최대 높이 제한 및 내부 스크롤 적용.

## 5. 향후 로드맵
- **MutationObserver 도입:** AJAX를 통한 테이블 갱신 시 티어 아이콘이 증발하는 현상 방어.
- **랜덤 디펜스:** 현재 필터링된 문제 중 하나를 무작위로 뽑아 바로 이동하는 기능.
- **스트릭/EXP:** 연속 풀이 일수 및 난이도 기반 경험치 시스템 시각화.
