# 🐛 버그 수정 보고서: Problemset 검색 버튼 클릭 이벤트 누락

## 1. 버그 증상 (Symptom)
* `Problemset` (전체 문제) 페이지 상단에 렌더링된 통합 검색창에서, 사용자가 텍스트를 입력하고 우측의 **[검색] 버튼을 마우스로 클릭해도 아무런 동작이 발생하지 않음.**
* 키보드 `Enter` 키나 쉼표(`,`)를 통한 알약(Pill) 생성은 정상적으로 동작함.

## 2. 원인 분석 (Root Cause)
* UI 뼈대(DOM)를 생성할 때 `<button id="boj-search-btn">검색</button>` 요소를 화면에 삽입하였음.
* 그러나 자바스크립트 로직 부에서 입력창(`input`)에 대한 키보드 이벤트(`keyup`, `keydown`)만 바인딩하고, **검색 버튼 요소에 대한 마우스 클릭(`click`) 이벤트 바인딩 로직을 작성하지 않고 누락함.**

## 3. 해결 및 수정 방안 (Resolution Strategy)
이벤트 누락을 해결하고 검색 버튼의 본래 목적(검색 실행)을 달성하기 위해 다음과 같이 로직을 추가 설계합니다.

### ① 클릭 이벤트 바인딩 추가
* 자바스크립트에서 검색 버튼 요소를 선택한 뒤, 마우스 `click` 이벤트 리스너를 명시적으로 연결합니다.

### ② 콜백(Callback) 동작 정의
버튼이 클릭되었을 때 다음 두 가지 동작이 순차적으로 실행되도록 로직을 구성합니다.
1. **잔여 텍스트 처리:** 입력창(Input)에 사용자가 텍스트를 적어두고 `Enter`를 치지 않은 상태로 [검색] 버튼을 누를 수 있습니다. 이 경우, 입력창에 남아있는 텍스트를 먼저 읽어와서 **알약(Pill)으로 강제 변환**한 뒤 입력창을 비워줍니다.
2. **검색(필터링) 실행 트리거:** 현재 화면에 생성되어 있는 모든 알약(Pill)들의 텍스트(조건)를 배열로 수집한 뒤, 이를 바탕으로 테이블을 걸러내는 **실제 필터링 엔진(AST 파서 및 렌더러)을 호출**합니다.

## 4. 기대 효과
* 키보드 타이핑에 익숙하지 않은 사용자도 마우스 클릭만으로 직관적인 검색 결과를 얻을 수 있습니다.
* 키보드(Enter)를 눌렀을 때와 마우스(Click)를 눌렀을 때의 동작 파이프라인이 하나로 통일되어 시스템 안정성이 높아집니다.
