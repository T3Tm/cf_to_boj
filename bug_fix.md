# 🐛 Bug Fix Report (v3.3.6)

## 1. 제출 실패 및 로직 정밀 보정 (Submission Execution Fix)
- **증상:** '제출' 버튼 클릭 후 채점 현황에 기록이 남지 않거나 실제 제출이 이루어지지 않음.
- **원인:** `originalForm.submit()` 메서드는 폼의 기본 `onsubmit` 핸들러(코드포스의 유효성 검사 및 데이터 가공 로직)를 거치지 않고 서버로 즉시 전송될 수 있음. 이로 인해 필수 데이터가 누락되거나 가공되지 않은 상태로 전송됨.
- **해결:** `submit()` 대신 원본 폼 내의 **실제 제출 버튼(`[type="submit"]`)을 `click()` 이벤트로 트리거**하여 코드포스의 모든 기본 제출 전처리가 정상 실행되도록 수정.

## 2. 자동 리다이렉션 (Automatic Redirection)
- **증상:** 제출 성공 후 의도한 페이지로 이동하지 않음.
- **원인:** 기존 방식은 브라우저의 기본 처리를 따름.
- **해결:** 제출 버튼 클릭 직전, 원본 폼의 데이터가 완벽히 채워졌는지 3차 검토하고, 제출 후 자연스럽게 채점 현황 페이지(`/problemset/status?contestId=...&index=...`)로 이동하도록 유도 (코드포스 기본 동작이 제출 후 현황판으로 가므로 이를 보장).

## 3. 잔여 중복 요소 제거 (Redundant Elements Cleanup)
- **증상:** "Submit solution", "Codeforces" 등 기존 레이아웃의 잔재가 페이지 상단에 노출됨.
- **원인:** `#pageContent` 내부의 특정 요소들만 처리하고, 그 외부나 상단 섹션을 놓침.
- **해결:** 페이지 상단의 `h2` 타이틀 및 불필요한 안내 텍스트 섹션을 CSS 선택자로 정밀 타겟팅하여 제거.
