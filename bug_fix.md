# 🐛 Bug Fix Report (v3.8.0)

## 1. 채점 현황 페이지네이션 (20개씩 보기) 구현
- **요구사항:** 한 페이지에 너무 많은 결과가 노출되지 않도록 20개 단위로 끊어서 표시.
- **해결:** 
    - 클라이언트 사이드 페이징 로직을 도입하여 로드된 데이터 중 20개만 노출하고 나머지는 `display: none` 처리함.
    - 하단에 '이전', '다음' 버튼과 현재 페이지 번호를 표시하는 백준 스타일 페이지네이션 바를 추가함.
    - 페이지 이동 시 스크롤을 최상단으로 이동시켜 사용자 편의성을 높임.

## 2. 문제 페이지 내 CF 원본 테이블 노출 버그 수정
- **증상:** 최근 CSS 업데이트 이후 문제 페이지 우측 사이드바나 하단에 코드포스 원본 테이블이 다시 보이기 시작함.
- **원인:** 채점 현황 테이블의 가시성을 위해 완화했던 CSS 규칙이 문제 페이지의 다른 테이블들까지 영향을 미침.
- **해결:** 
    - `overrides.css`에서 `#pageContent .datatable`에 대한 전역 스타일을 더 구체적으로 정의함.
    - 채점 현황 페이지가 아닐 경우 원본 데이터 테이블을 더 엄격하게 숨기도록 규칙을 강화함.

## 3. 동적 업데이트 및 페이징 상태 유지
- **해결:** `DOMObserver`가 테이블 갱신을 감지해도 현재 사용자가 보고 있는 페이지 번호(`currentPage`)가 유지되도록 로직을 설계함.
