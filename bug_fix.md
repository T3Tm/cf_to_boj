# 🐛 Bug Fix Report (v3.6.0)

## 1. 필터링 로직 전면 개편 (문제별 현황 및 내 제출)
- **증상:** 문제 페이지에서 '채점 현황'으로 이동 시 필터링이 작동하지 않거나, '내 제출' 클릭 시 결과가 바뀌지 않음.
- **원인:** 
    - 코드포스 문제셋 현황은 쿼리 파라미터(`?contestId=...`)가 아닌 경로 기반(`/status/ID/problem/INDEX`) 필터링을 주로 사용함.
    - `handle` 파라미터 토글 시 기존 필터 경로가 유실되어 전체 결과로 초기화되는 현상 발생.
- **해결:** 
    - `src/pages/problem.js`의 링크 형식을 경로 기반으로 변경함.
    - `src/pages/status.js`에서 경로(`pathname`)와 쿼리(`search`)를 모두 분석하여 현재 필터 상태를 정확히 감지하도록 개선함.
    - `URLSearchParams`를 사용하여 기존 필터를 유지하며 `handle`만 토글하도록 수정함.

## 2. 백준 스타일 채점 현황 UI 완성 (Perfect Clone)
- **증상:** UI가 여전히 코드포스 고유의 스타일을 유지하여 이질감이 느껴짐.
- **해결:** 
    - `overrides.css`에 백준 특유의 회색 헤더 배경(`#f9f9f9`), 상단 파란색 보더(`#0076c0`), 줄무늬 행 스타일을 추가함.
    - 테이블 내 영문 텍스트(Accepted, Wrong Answer 등)를 '맞았습니다!!', '틀렸습니다' 등 백준 용어로 한국어 번역함.
    - 아이디와 문제명 컬럼을 왼쪽 정렬하고 패딩을 조절하여 백준과의 시각적 동질성을 확보함.

## 3. 강력한 요소 클리너 및 가시성 확보
- **증상:** "My only", "Friends only" 등 네이티브 스위치가 사라지지 않거나 테이블이 숨겨짐.
- **해결:** 
    - `friendsEnabledSwitch`, `myEnabledSwitch` 등 특정 클래스를 가진 컨테이너와 floating div를 명시적으로 제거함.
    - CSS `!important` 의존도를 낮추기 위해 JS에서 `style.setProperty`를 사용하여 필수 가시성 요소만 정밀하게 제어함.
    - `DOMObserver`를 통해 동적 갱신 시에도 클린업 로직이 상시 작동하도록 설계함.
