# 🏆 Codeforces to Baekjoon (v3.20 Sorting Integrity & Routing Hotfix)

본 명세서는 쿼리 필터링과 테이블 정렬 기능 간의 충돌(Closure Trap)을 해결하고, 불필요한 헤더 메뉴 삭제 및 타 페이지 라우팅 경로를 코드포스 규격에 맞게 정상화하는 프론트엔드 핫픽스 지침서입니다.

---

## 🚨 1. 3회 교차 검증 및 치명적 버그 원인 분석 (3-Pass Debugging)

### ① 1차 검증: 정렬 클릭 시 쿼리(알약)가 무시되고 전체 목록이 뜨는 이유
* **원인 규명 (클로저 함정):** 이전 버전에서 가상 테이블(`boj-virtual-table`) 컨테이너를 최초 1회 생성할 때 클릭 이벤트(Event Delegation)를 달아주었습니다. 이때 클릭 이벤트 내부에서 `buildVirtualTable(problems)`를 호출하도록 설계했습니다. 자바스크립트의 특성상, 이 이벤트 리스너는 **최초로 테이블이 만들어질 때의 `problems` 배열(즉, 필터링되지 않은 9000개의 전체 문제 배열)을 메모리에 영구적으로 기억(포획, Closure)**해버립니다.
* **결과:** 이후에 알약을 입력해서 10개짜리 필터링된 배열을 화면에 그렸더라도, 헤더를 클릭하는 순간 렌더링 엔진은 기억해둔 '9000개짜리 원본 배열'을 꺼내와서 정렬하고 렌더링해버린 것입니다.
* **철벽 방어 설계:** 이벤트 리스너 안에서 '기억된 배열'을 사용하는 낡은 방식을 버려야 합니다. 정렬 헤더를 클릭하면, 단순히 정렬 모드(`sortMode`)와 방향(`sortAsc`) 변수만 변경한 직후, **상태 관리자(StateManager)에게 "현재 활성화된 알약 상태를 기준으로 처음부터 다시 필터링해서 그려라!"라고 지시(`handleFilters` 함수 재호출)**하도록 파이프라인의 물꼬를 돌려야 합니다.

### ② 2차 검증: 상단 바 '제출' 메뉴의 불필요성
* **원인 규명:** 코드포스는 개별 문제 페이지에 진입해야만 코드를 제출할 수 있는 구조(`problemset/submit/문제번호`)입니다. 따라서 전역 헤더(GNB)에 '제출' 버튼이 덩그러니 있는 것은 논리적으로 맞지 않으며 공간만 차지합니다.
* **해결 설계:** 메인 자바스크립트 파일에서 새로운 헤더를 문자열로 조립할 때 삽입했던 '제출' 항목(li 태그)을 완전히 삭제하여 GNB를 직관적으로 다이어트시킵니다.

### ③ 3차 검증: 상단 바 라우팅(이동)이 원활하지 않은 이유
* **원인 규명:** 새로 만든 헤더의 링크(`href`) 주소들이 코드포스의 실제 기본 라우팅 경로와 미세하게 어긋나 있거나 리다이렉션을 유발하고 있었습니다. (예: 랭킹 페이지는 `/top`이 아니라 `/ratings`가 표준 정적 경로입니다.)
* **해결 설계:** 상단 바의 링크 주소를 코드포스의 표준 정적 URL로 전면 교체합니다.
  - 문제: `/problemset`
  - 대회: `/contests`
  - 채점 현황: `/problemset/status`
  - 랭킹: `/ratings`
  - 그룹: `/groups`

## 🛠️ 2. 아키텍처 핫픽스 조치 지침 (Implementation Guide)

개발자님께서는 코드를 직접 수정하실 때 아래의 3가지 지침을 엄격하게 따라 주십시오. (코드는 제공하지 않으므로 논리 흐름에 따라 직접 개편을 진행하셔야 합니다.)

### 지침 1: `src/pages/problemset.js` 정렬 이벤트 롤백 방지 (핵심)
1. `buildVirtualTable` 함수 내부에서 가상 테이블에 `addEventListener('click', ...)`를 달아주는 부분을 찾으십시오.
2. 페이지 번호를 클릭하거나, 정렬 헤더(<th>)를 클릭했을 때 모두 `buildVirtualTable(problems)`를 직접 호출하고 있는 부분을 찾으십시오.
3. 이 직접 호출 코드를 모두 지우고, 대신 **`handleFilters(window.BOJ_CF.StateManager.getState())`** 를 호출하도록 수정하십시오.
4. 이렇게 변경하면 정렬/페이지 변경 시 현재 유저가 만들어둔 알약(State)을 읽어와 다시 필터링을 거친 후 현재 뷰만 정확하게 정렬하여 렌더링하게 됩니다.

### 지침 2: `main.js` 헤더 '제출' 메뉴 철거
1. `main.js` 최상단에서 `innerHTML`을 사용해 `boj-header-center` 영역을 생성하는 코드를 찾으십시오.
2. 목록(ul) 안에 있는 **'제출'** 링크 줄을 찾아 통째로 삭제하십시오.

### 지침 3: `main.js` 헤더 라우팅 경로 표준화
1. '제출' 메뉴를 지운 바로 그 영역에서, 나머지 링크들의 `href` 속성을 점검하십시오.
2. 랭킹의 주소를 `/top`에서 **`/ratings`**로 변경하십시오.
3. 그룹의 주소를 `/groups/my`에서 범용적인 **`/groups`**로 변경하십시오.
4. 이제 헤더 버튼을 클릭했을 때 자바스크립트 충돌이나 리다이렉션 지연 없이 즉각적으로 페이지가 이동될 것입니다.
