# 🛠️ Codeforces 백준화 프로젝트: 향후 업데이트 및 버그 수정 명세서

본 문서는 현재 발생한 레이아웃 충돌 해결 방안과, 향후 코드포스 환경에서 안정적인 동작을 보장하기 위해 도입해야 할 핵심 기능들의 설계 명세입니다.

## 1. 긴급 수정 사항 (Hotfix)

### ① Problemset 상단 레이아웃 충돌 해결 (UI/CSS)
* **문제 상황:** 새롭게 추가된 '통합 검색창(Search UI)' 영역과 코드포스 본연의 '상단 페이지네이션(Pagination)' 및 '문제 목록 테이블(Table)'이 위아래로 겹치는(Overlap) 현상 발생.
* **원인:** 코드포스 기존 요소들의 `float` 속성 유지 및 명확한 블록 분리(`clear`) 부재.
* **해결 방안:**
  1. **상단 페이지네이션 제거:** 테이블 위의 불필요한 페이지 이동 버튼을 숨김 처리(`display: none`)하여 공간 확보. (하단 페이지네이션으로 대체)
  2. **블록 분리 및 여백 확보:** 검색창 컨테이너 하단에 충분한 `margin-bottom`을 부여하고, CSS에 `clear: both` 속성을 적용하여 테이블이 검색창 영역을 침범하지 못하도록 강제 차단.

## 2. 향후 업데이트 필수 사항 (Core Updates)

### ① AJAX 통신 방어 로직 (MutationObserver 도입)
* **문제 상황 (예측):** 사용자가 표 상단의 정렬(↕) 화살표를 누르거나, 다른 페이지(2페이지 등)로 이동할 때 화면 전체가 새로고침되지 않음.
* **원인:** 코드포스는 표 부분만 서버에서 비동기(AJAX)로 받아와서 덮어씌움. 이 과정에서 우리가 삽입한 티어 아이콘(`img`)과 텍스트 번역이 모두 초기화(증발)됨.
* **해결 방안:** * 자바스크립트의 `MutationObserver` API를 사용하여 테이블 DOM 요소의 변화를 실시간으로 감시.
  * 테이블 내용이 갱신되는 순간을 포착하여, 0.01초 이내에 `Utils.getProblemTierIcon` 등 렌더링 함수를 재실행시켜 아이콘과 UI를 복구(Re-render).

### ② 검색 UI 반응형 및 안정성 강화
* **알약(Pill) 컨테이너 제한:** 검색 조건이 너무 많아져 알약이 과다 생성될 경우, 메인 레이아웃을 밀어내지 않도록 `max-height`와 내부 스크롤(`overflow-y: auto`) 적용.
* **반응형(Responsive) 방어:** 브라우저 창 크기가 작아질 때 검색바와 검색 버튼의 비율이 깨지지 않도록 Flexbox 속성(`flex-wrap`, `min-width`)을 정교하게 튜닝.

## 3. 심화 기능 구현 계획 (Feature Roadmap)

### ① 단일 페이지 애플리케이션(SPA) 기반 필터링 엔진
* **개요:** 코드포스 서버에 검색 쿼리를 보내지 않고, 브라우저 단에서 100% 독립적으로 문제를 필터링하는 시스템 구축.
* **구현 로직:**
  1. `problemset.problems` API를 호출하여 전체 문제 메타데이터(~9,000개)를 브라우저 메모리에 캐싱.
  2. AST(추상 구문 트리) 기반의 커스텀 파서를 통해 유저가 입력한 복합 쿼리(예: `~@me & (#dp | #math) & *s5..g5`)를 해독.
  3. 메모리에 있는 문제 배열을 조건에 맞게 필터링(Array.filter)한 뒤, 화면의 테이블 DOM을 직접 조작하여 렌더링.

### ② 나의 상태(Status) 기반 필터 고도화
* **기능:** `status:solved`, `status:unsolved`, `status:tried` 등의 쿼리 지원.
* **구현 로직:** 확장 프로그램 로드 시 `user.status` API를 백그라운드에서 1회 호출하여 현재 로그인한 유저의 제출 기록(OK 및 기타 판정)을 Set 자료구조에 미리 매핑해 두고, 검색 시 이를 대조하는 방식으로 지연 없는 필터링 제공.
