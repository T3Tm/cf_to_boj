## 🚨 1. 치명적 버그 원인 규명 및 철벽 방어 설계 (Deep Debugging)

### ① 테마 전환 및 검색 버튼 먹통의 진짜 이유 (DOM Lifecycle Mismatch)
* **원인 규명:** 자바스크립트는 `innerHTML`로 요소를 그리는 속도와, 그 요소를 찾아서 이벤트(`onclick`)를 달아주는 속도 사이에 미세한 오차가 발생할 수 있습니다. 특히 헤더와 검색창을 새로 주입하는 과정에서 렌더링 타이밍이 꼬여, 버튼은 화면에 보이지만 이벤트 리스너는 허공에 등록되는 **'좀비 버튼(Zombie Element)'** 현상이 발생했습니다.
* **철벽 방어 (Event Delegation):** 요소를 찾아서 이벤트를 다는 구시대적 방식을 전면 폐기합니다. 브라우저의 최상위 객체인 `document.body`에 딱 하나의 클릭 이벤트를 달고, "클릭된 녀석의 아이디가 `boj-theme-toggle` 이면 테마를 바꿔라", "`boj-search-btn` 이면 검색을 실행해라" 라고 지시하는 **이벤트 위임(Event Delegation)** 패턴으로 아키텍처를 교체합니다. 이 방식을 사용하면 요소가 언제, 어떻게 렌더링되든 100% 완벽하게 이벤트가 작동합니다.

### ② 쿼리 검색 시 진행이 멈추는 이유 (Reference Error Halt)
* **원인 규명:** 검색창에 'dp'를 치고 엔터를 눌렀을 때 아무 반응이 없는 이유는, 화면을 갱신하는 파이프라인 중간에 자바스크립트 에러가 발생해 실행이 강제 중단(Halt)되었기 때문입니다. 이전 명세에서 알약 컨테이너(`PillContainer`)를 수정할 때 변수명을 혼용(`containerElement` vs `container`)하여 발생한 **참조 에러(ReferenceError)**가 전체 검색 프로세스를 마비시켰습니다.
* **철벽 방어:** 상태 관리자(`StateManager`)와 `PillContainer` 간의 변수 스코프를 엄격하게 일치시키고, DOM 객체를 변수에 담아두지 않고 렌더링 시점에 실시간으로 쿼리(Query)하여 덮어쓰도록 안전장치를 재구축합니다.

### ③ 초기 접속 시 테이블이 비어있는 이유 (API Rate Limit 503 Error)
* **원인 규명:** 스크린샷에서 검색 결과가 없다고 나오는 이유는, 내부 데이터베이스(`globalDB`)가 텅 비었기 때문입니다. 왜 비었을까요? 코드포스 API는 **'1초에 1회 요청'**이라는 엄격한 호출 제한(Rate Limit)이 있습니다. 그런데 기존 로직은 `Promise.all`을 사용해 전체 문제 API와 유저 정보 API를 **0.001초 만에 동시에 공격하듯 호출**했습니다. 그 결과 서버가 요청을 차단(HTTP 503)했고, 에러를 반환받아 데이터베이스가 구축되지 않은 것입니다.
* **철벽 방어 (API Serialization):** 비동기 병렬 호출(`Promise.all`)을 폐기합니다. 전체 문제(`fetchAllProblems`)를 먼저 가져오고 완전히 끝난 것을 확인한 뒤에야, 순차적으로 유저 정보(`fetchUserStatus`)를 가져오도록 직렬화(Serialization)하여 서버의 차단을 완벽하게 우회합니다.

## 🛠️ 2. 아키텍처 추가 고도화 기획 (BOJ Details)

버그 수정을 넘어, 백준의 사용자 경험(UX)을 완벽하게 재현하기 위한 2가지 디테일 고도화 설계입니다.

### 디테일 1: 일반 텍스트의 '태그 자동 검색' 지원 (Smart Fallback)
* **문제점:** 유저가 검색창에 `tag:dp`가 아닌 그냥 `dp`라고 쳤을 때, 기존 파서는 문제의 '제목(Name)'에서만 dp를 찾았습니다. 제목에 dp가 들어간 문제는 거의 없으므로 결과가 나오지 않았습니다.
* **해결책:** 파서 엔진을 업그레이드하여, 특정 접두사(`tag:`, `r:`)가 없는 일반 텍스트를 입력할 경우 **'문제 제목' 또는 '태그 배열' 중 하나라도 일치하면 검색되도록** 스마트 폴백(Smart Fallback) 로직을 탑재합니다.

### 디테일 2: '맞힌 사람' 열 추가 및 클릭 정렬 기능 (Sorting by Solved)
* **기능 설계:** 백준처럼 '국민 문제'부터 차례대로 풀 수 있도록, 가상 테이블 헤더의 `<th>맞힌 사람</th>` 텍스트에 정렬 토글 기능을 부여합니다. 클릭할 때마다 레이팅 오름차순 정렬에서 정답자 수 내림차순 정렬로 스위칭되도록 로컬 상태 변수(`sortMode`)를 도입합니다.
