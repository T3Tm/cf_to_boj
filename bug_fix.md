# 🏆 Codeforces to Baekjoon (v3.8 UI/UX Grand Remaster)

본 명세서는 검색 엔진의 기능을 넘어, 시각적인 레이아웃과 네비게이션, 그리고 데이터 정렬 방식까지 완벽하게 백준(BOJ) 스타일로 동기화하기 위한 프론트엔드 아키텍처 대공사 지침서입니다. 모든 설계는 하드코딩을 배제하고 설정(Config) 기반으로 확장 가능하도록 구성되었습니다.

---

## 🚨 1. 3회 교차 검증 및 아키텍처 분석 (Verification)

### ① 1차 검증: 문제 난이도 오름차순 정렬 누락
* **원인:** API(`problemset.problems`)는 기본적으로 가장 최근에 열린 대회(Contest ID 역순)부터 데이터를 반환합니다. 이 데이터를 그대로 가상 테이블에 렌더링했기 때문에 난이도가 뒤죽박죽이었습니다.
* **해결 설계:** 렌더링 파이프라인 중간에 **'정렬(Sorting) 미들웨어'**를 추가합니다. 단순히 점수(Rating)를 빼는 하드코딩 로직이 아니라, 정렬 기준(Key)과 방향(Asc/Desc)을 인자로 받아 데이터를 정렬하는 범용 유틸리티 함수를 거친 뒤 상위 100개를 잘라내도록 데이터 흐름을 개조합니다. (레이팅이 없는 문제는 최하단으로 밀어냅니다.)

### ② 2차 검증: 백준 느낌의 상실 (레이아웃 잔재)
* **원인:** 백준의 핵심 레이아웃은 **'사이드바가 없는 중앙 정렬된 1단 넓은 레이아웃'**입니다. 하지만 현재 문제 목록 페이지에는 코드포스 우측의 `Pay attention`이나 `Filter` 같은 거대한 사이드바가 그대로 남아있어 시선이 분산되고 공간이 좁아졌습니다.
* **해결 설계:** 개별 문제 페이지(`problem.js`)에서 했던 것처럼, 문제 목록 컨트롤러(`problemset.js`)가 실행될 때도 DOM에서 `#sidebar`를 완전히 숨김(display: none) 처리하고, 메인 콘텐츠 영역을 중앙으로 넓게 확장하는 레이아웃 제어 로직을 공통화하여 주입합니다.

### ③ 3차 검증: GNB(상단바)와 테이블 디자인의 불일치
* **원인:** 백준의 상단바는 흰색 배경에 파란색 로고, 깔끔한 텍스트 메뉴로 이루어져 있습니다. 테이블 역시 상단에 두꺼운 파란색 선(`border-top: 2px solid blue`)이 들어간 심플한 형태입니다.
* **해결 설계:** 하드코딩된 색상을 지양하기 위해 CSS 변수(`--boj-primary-color`)를 도입합니다. 코드포스의 복잡한 로고 영역과 메뉴 영역을 Flexbox 레이아웃으로 재정의하여 사진 4번과 같이 좌측 로고, 중앙 메뉴, 우측 유저 정보로 일렬 배치되도록 CSS를 전면 개편합니다.

## 🛠️ 2. 프론트엔드 개편 지침 (Implementation Guide)

개발자님께서는 아래 4단계 지침에 따라 기존 로직과 CSS를 유연한 구조로 재작성해 주십시오.

### Step 1. 유연한 정렬 파이프라인 구축 (문제 오름차순)
* **대상:** `pages/problemset.js` 내부의 렌더링 로직 (`handleFilters` 또는 가상 테이블 생성 직전)
* **지침:**
  1. 데이터를 렌더링하기 직전, 전체 데이터 배열(`globalDB`)이나 검색된 배열(`filtered`)을 정렬하는 범용 함수를 호출하십시오.
  2. 기준은 `rating`으로 하되, 오름차순(Ascending)으로 정렬하십시오.
  3. **예외 처리:** 레이팅이 없는 문제(`undefined` 또는 `0`)는 수학적 계산 시 에러를 유발하거나 맨 위로 올라올 수 있으므로, 방어 로직을 짜서 무한대(Infinity) 값으로 취급하여 배열의 가장 맨 끝으로 안전하게 밀어내십시오.

### Step 2. 1단 중앙 레이아웃 강제화 (사이드바 철거)
* **대상:** `pages/problemset.js` 초기화(`init`) 로직
* **지침:**
  1. 페이지 로드 직후 `document.getElementById('sidebar')`를 찾아 `display: none`으로 화면에서 지우십시오.
  2. 메인 컨테이너(`#pageContent` 또는 래퍼 클래스)의 `float` 속성을 해제하고, `width: 100%`, `max-width: 1200px` (사진 4번과 유사한 넓이), `margin: 0 auto`를 부여하여 화면 정중앙에 배치하십시오.

### Step 3. 상단 GNB(Global Navigation Bar) 모던화
* **대상:** `styles/overrides.css`
* **지침 (사진 4번 구현):**
  1. 상단 헤더 컨테이너를 `display: flex`, `align-items: center`, `justify-content: space-between`으로 만들어 좌/중/우 배치를 강제하십시오.
  2. 메뉴 이름(HOME, TOP 등)을 한글(문제, 대회, 채점 현황, 랭킹)로 바꾸는 것은 JS로 DOM의 텍스트를 치환(replace)하는 유틸리티 함수를 만들어 `main.js`에서 실행하십시오. (하드코딩 방지: 영어-한글 매핑 객체 활용)
  3. 헤더의 배경을 투명 또는 흰색으로 빼고, 하단에만 얇은 회색 선을 그어 백준의 플랫한 헤더를 재현하십시오.

### Step 4. 테이블 UI 백준 동기화
* **대상:** `styles/components.css` 및 `pages/problemset.js` (가상 테이블 생성부)
* **지침:**
  1. **컬럼명 변경:** 가상 테이블 생성 시 하드코딩된 `#`, `Name` 대신, `문제`, `제목`, `난이도`, `✅(아이콘)`으로 테이블 헤더 문구를 수정하십시오.
  2. **테이블 CSS:** 테이블 헤더(`th`) 영역 상단에 파란색 두꺼운 테두리(`border-top: 2px solid var(--boj-primary);`)를 추가하십시오.
  3. 테이블 셀(`td`, `th`)의 좌우 테두리(세로선)는 모두 지우고(`border-left/right: none`), 상하 테두리(가로선)만 남겨 백준 특유의 개방감 있는 표 디자인을 완성하십시오.
  4. 행(Row)에 마우스를 올렸을 때의 호버(Hover) 색상도 아주 연한 파란색이나 회색으로 지정하여 인터랙션을 살리십시오.

## 📝 3. 시스템 안정성 및 유지보수 점검 (Checklist)

* [ ] **Anti-Hardcoding:** 정렬 기준(rating)이나 방향(asc)을 함수 내부에서 매직 넘버로 빼지 않고, 매개변수나 설정 객체로 분리하여 나중에 '내림차순 정렬' 버튼을 쉽게 추가할 수 있도록 설계했는가?
* [ ] **Layout Safety:** 사이드바를 지움으로써 발생할 수 있는 레이아웃 붕괴 현상이 없는가? 화면 크기를 줄였을 때(반응형) 중앙 정렬이 유지되는가?
* [ ] **Sorting Integrity:** 검색창에 아무것도 입력하지 않은 기본 상태에서도 800점짜리(가장 쉬운) 문제부터 차례대로 표출되는가?
* [ ] **Design Match:** 테이블 상단에 굵은 파란색 선이 생겼으며, 컬럼명들이 한글로 명확히 표시되는가?
