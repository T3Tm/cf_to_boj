# 🐛 Bug Fix Report (v3.5.0)

## 1. 채점 현황 데이터 로드 및 테이블 표시 무결성 확보
- **증상:** 일부 채점 현황 페이지에서 테이블 정보가 표시되지 않거나, 헤더 번역 및 티어 아이콘 주입이 누락됨.
- **원인:** 
    - 특정 경로(contest 등)에서 테이블 선택자가 엄격하여 요소를 찾지 못함.
    - CSS의 `display: none !important` 규칙이 데이터가 로드된 후에도 테이블 가시성을 방해함.
- **해결:** 
    - `table.status-frame-datatable` 외에도 `table.datatable` 등 포괄적인 선택자를 지원하도록 개선.
    - JS 실행 시 테이블 및 부모 컨테이너의 `display` 속성을 강제로 제어하여 가시성을 보장함.
    - 헤더 매핑 시 `startsWith`를 사용하여 'Sent', 'Submit Time' 등 유사 헤더를 모두 '제출 시간'으로 통합 번역함.

## 2. 문제별 vs 전체 채점 현황 라우팅 및 UI 분리
- **증상:** 상단바를 통해 들어온 전체 현황에서도 문제 탭 메뉴가 남거나, 반대로 문제 페이지에서 들어왔을 때 탭 메뉴가 실종됨.
- **해결:** `URLSearchParams`를 통해 `contestId`와 `index`의 존재 여부를 1단계로 검토하여 UI 모드(isFromProblem)를 명확히 구분함.

## 3. 네이티브 필터 잔재 제거
- **증상:** "My only", "Friends only" 등 기존 코드포스의 링크가 백준 스타일 UI와 섞여 가독성을 해침.
- **해결:** `cleanupNativeElements` 함수를 통해 관련 텍스트 링크 및 컨테이너를 영구 삭제함.

## 4. 3단계 자가 검토 프로세스 도입
- **수행:** 1) URL/상태 분석, 2) UI 초기화 및 잔재 제거, 3) 데이터 변환 및 안정성 확보 순으로 로직을 재구성하고 각 단계별 예외 처리를 강화함.
