# 🐛 Bug Fix Report (v3.4.3)

## 1. 제출 버튼 동작 불능 해결 (Form Context Integrity)
- **증상:** '제출' 버튼을 눌러도 아무런 반응이 없거나 코드포스 서버에서 거부됨.
- **원인:** 입력 요소와 버튼을 원본 `<form>` 태그 외부로 이동시킴으로써 폼 컨텍스트가 깨짐.
- **해결:** 원본 `<form>` 태그를 그대로 유지한 채 내부 시각적 요소들만 재배치하여 로직 무결성을 확보함.

## 2. 채점 현황 테이블 실종 및 시각적 노이즈 해결
- **증상 1:** 채점 현황 페이지에서 테이블이 아예 보이지 않음.
- **증상 2:** "My only", "Friends only" 등 코드포스 네이티브 필터 텍스트가 남아있어 UI가 지저분함.
- **원인:** CSS의 `display: none` 규칙이 채점 현황 테이블까지 과도하게 적용됨.
- **해결:** 
    - `overrides.css`의 테이블 숨김 예외 규칙에 `[class*="status"]`, `[id*="status"]`를 추가하여 채점 현황 테이블의 가시성을 강제 확보함.
    - `src/pages/status.js` 내에 `cleanupNativeElements` 로직을 추가하여 네이티브 필터 링크 및 잔재 메뉴들을 완전히 삭제함.

## 3. 필터 연속성 및 로그인 예외 처리
- **증상:** 비로그인 상태에서 '내 제출' 필터를 누를 시 오동작 가능성.
- **해결:** 로그인 상태(`isLoggedIn`)를 체크하여 비로그인 시 체크박스를 비활성화(`disabled`)하고 안내 문구를 추가함. 또한 URLSearchParams를 통해 기존 필터 상태를 유지하며 동작하도록 개선함.
