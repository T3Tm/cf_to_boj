# 🐛 Bug Fix Report (v3.3.5)

## 1. 제출 버튼 작동 오류 (Submission Logic Failure)
- **증상:** '제출' 버튼 클릭 시 코드포스의 실제 제출 로직이 실행되지 않거나 오류 페이지가 표시됨.
- **원인:** 커스텀 폼(`boj-submit-form`)을 새로 구축하면서 원본 폼이 기대하는 `onSubmit` 이벤트 핸들러나 숨겨진 검증 필드가 누락되었거나, 폼 제출 시 필요한 특정 데이터가 정확히 전달되지 않음.
- **해결:** 커스텀 폼을 직접 제출하는 대신, 사용자가 커스텀 폼의 제출 버튼을 누르면 **데이터를 원본 폼에 동기화한 뒤 원본 폼을 트리거**하는 방식으로 우회하여 코드포스의 자체 보안 및 제출 로직을 보존함.

## 2. 탭 메뉴 가운데 정렬 수정 (Tab Menu Alignment)
- **증상:** 제출 페이지의 탭 메뉴가 가운데로 정렬되어 시각적 불균형 발생.
- **원인:** `submit.js`에서 탭 메뉴에 강제로 `justify-content: center`를 적용했음.
- **해결:** 탭 메뉴의 인라인 스타일을 제거하고 `overrides.css`의 글로벌 정렬 기준(왼쪽 정렬)을 따르도록 수정하여 백준 스타일과의 정합성을 맞춤.

## 3. 폼 중복 제거 로직 강화
- **증상:** 일부 환경에서 여전히 원본 요소가 슬쩍 보이는 현상.
- **원인:** `init` 실행 시점과 코드포스의 동적 렌더링 시점의 충돌.
- **해결:** `MutationObserver`는 이미 존재하므로, `init` 호출 시 `roundbox`를 완전히 부모 노드에서 `remove()` 처리하여 노출 가능성을 차단함.
